# Submission Checklist

## Before Coding
- Confirm the exact topic chosen from the course spreadsheet
- Save a photo showing the topic selection date and time
- Confirm all team members and module ownership
- Agree on port names, mode values, and signal priority

## Before GitHub Submission
- All source files are uploaded
- `README.md` is complete
- File names are correct
- The project simulates without missing files

## Before Final Report
- Add the GitHub link at the beginning
- Add names, student IDs, and signatures
- Add the project description
- Add the design schematic
- Add Verilog code
- Add GTKWave screenshots
- Add testbench results
- Keep the report within 6 pages

## Before ALMS Submission
- Put source code and report in one folder
- Compress them into a single ZIP file
- Upload the ZIP file to ALMS
- Make sure every group member uploads the same file to ALMS

## Before Presentation
- Prepare a 2-minute explanation
- Prepare one member to present on Wednesday, May 06, 2026
- Bring one printed and signed report
